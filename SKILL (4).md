# Journal

One file per day, named `YYYY-MM-DD.md`. This is the vault's short-term memory — what happened, what got decided, what's next. `capture-everything` writes here automatically during and after sessions.

## Template

```markdown
# YYYY-MM-DD

## What happened today
-

## Decisions made
-

## New people / clients encountered
-

## Tomorrow / next steps
-
```

Multiple sessions in one day get a `### Session: [time or topic]` sub-heading so they don't blur together. Wrap entity names (people, clients, projects) in `[[double brackets]]` on first mention per section — this keeps Obsidian's graph and backlinks useful instead of decorative.
